> **3.1 签到地理围栏**

**功能概述**：将签到距离校验从"以活动点为圆心500m固定范围"升级为"组织者手绘任意多边形签到区域"，后端使用射线法（Ray Casting Algorithm）判定GPS坐标是否在围栏内。

**操作步骤（4步大功能）**：

（1）**进入围栏编辑**：管理员/组织者在活动列表点击"围栏"按钮，跳转到GeofenceEdit.vue页面。页面加载时调用`GET /api/activities/{id}/geofence`检查是否已有围栏，同时加载活动地点坐标，在地图上以蓝色标记点展示活动位置供围栏绘制参考。

（2）**绘制多边形**：点击"编辑签到围栏"按钮进入编辑模式。用户在地图上逐点点击添加多边形顶点，GeofenceEditor.vue通过MapLibre click事件收集顶点坐标（GCJ-02），实时渲染橙色虚线边框和顶点圆点。支持撤销（移除最后一个顶点）和清空操作。至少需要3个顶点才能完成绘制。

（3）**保存围栏**：点击"完成绘制"验证顶点数≥3，地图上实时显示橙色半透明填充面预览围栏效果。点击"保存围栏"将多边形顶点数组构造为GeoJSON Polygon字符串，调用`PUT /api/activities/{id}/geofence`存入activity.checkin_region字段。

（4）**围栏内签到校验**：学生签到时，CheckInServiceImpl.checkIn()方法优先检查activity.checkin_region。若存在围栏→调用parsePolygonFromGeoJson()解析→isPointInPolygon()射线法判定GPS坐标是否在多边形内。围栏外拒绝签到，无围栏则兜底使用Haversine公式500m圆形校验。

```java
// SpatialCalculator.java - 射线法判定点是否在多边形内
public boolean isPointInPolygon(double lng, double lat, double[][] polygon) {
    if (polygon == null || polygon.length < 3) return false;
    int n = polygon.length;
    boolean inside = false;
    // 从待测点向右发水平射线，统计与多边形边的交点数
    for (int i = 0, j = n - 1; i < n; j = i++) {
        double xi = polygon[i][0], yi = polygon[i][1];
        double xj = polygon[j][0], yj = polygon[j][1];
        // 射线与边(y=lat)相交的判定条件
        if ((yi > lat) != (yj > lat)
                && lng < (xj - xi) * (lat - yi) / (yj - yi) + xi) {
            inside = !inside;  // 交点 → 翻转内外状态
        }
    }
    return inside; // 奇数个交点=内部，偶数=外部
}
```

```java
// SpatialCalculator.java - GeoJSON Polygon字符串解析
public double[][] parsePolygonFromGeoJson(String geoJson) {
    if (geoJson == null || geoJson.isBlank()) return null;
    // 解析格式: {"type":"Polygon","coordinates":[[[lng1,lat1],[lng2,lat2],...]]}
    String content = geoJson.replaceAll(".*?\\[\\[\\[", "")
                             .replaceAll("\\]\\]\\].*", "");
    String[] pairs = content.split("\\],\\[");
    double[][] polygon = new double[pairs.length][2];
    for (int i = 0; i < pairs.length; i++) {
        String[] xy = pairs[i].trim().split(",");
        polygon[i][0] = Double.parseDouble(xy[0].trim());
        polygon[i][1] = Double.parseDouble(xy[1].trim());
    }
    return polygon;
}
```

> **3.2 热力图分析**

**功能概述**：基于signup表中历史签到坐标数据，使用MapLibre GL heatmap图层渲染志愿活动空间密度分布。支持按活动分类和时间范围筛选，帮助学生和管理员直观了解志愿活动高频区域和"服务盲区"。

**操作步骤（4步大功能）**：

（1）**打开热力图****：用户在地图页面右下角点击"🔥 活动热力图"按钮（渐变色卡片样式），展开热力图控制面板。面板包含分类下拉框和时间范围下拉框。

（2）**设置筛选条件**：在面板中选择活动分类（环保/助学/支教/社区/校园/全部）和时间范围（近3/6/12个月），选择器change事件自动触发loadHeatmap()重新请求数据。

（3）**加载热力数据**：前端调用`GET /api/map/heatmap?category=xxx&months=N`接口。后端：(a)计算since=N months ago；(b)若指定category则查出匹配activityIds过滤；(c)查询signup表中sign_in_lng/lat非空且sign_in_time≥since的记录；(d)调用CoordConvertService.wgs84ToGcj02()将WGS-84签到坐标转为GCJ-02；(e)返回GeoJSON Point FeatureCollection，weight属性基于volunteerHours，无时长数据默认0.5。

```java
// MapServiceImpl.java - 热力图数据查询与坐标转换
@Override
public FeatureCollection getHeatmapData(String category, int months) {
    LocalDateTime since = LocalDateTime.now().minusMonths(months);
    // 按分类筛选活动ID
    List<Long> activityIds = null;
    if (category != null && !category.isBlank()) {
        activityIds = activityMapper.selectList(
                new LambdaQueryWrapper<Activity>()
                        .eq(Activity::getCategory, category))
                .stream().map(Activity::getId).collect(Collectors.toList());
        if (activityIds.isEmpty()) return geoJsonBuilder.collection(List.of());
    }
    // 查询有签到坐标的记录
    LambdaQueryWrapper<Signup> wrapper = new LambdaQueryWrapper<>();
    wrapper.isNotNull(Signup::getSignInLng).isNotNull(Signup::getSignInLat)
           .ge(Signup::getSignInTime, since);
    if (activityIds != null) wrapper.in(Signup::getActivityId, activityIds);
    List<Signup> signups = signupMapper.selectList(wrapper);
    // 构造GeoJSON（WGS-84 → GCJ-02）
    List<Feature> features = new ArrayList<>();
    for (Signup s : signups) {
        double[] gcj = coordConvertService.wgs84ToGcj02(
            s.getSignInLng().doubleValue(), s.getSignInLat().doubleValue());
        Map<String, Object> props = geoJsonBuilder.props();
        props.put("activityId", s.getActivityId());
        props.put("weight", s.getVolunteerHours() != null
                ? s.getVolunteerHours().doubleValue() : 0.5);
        features.add(geoJsonBuilder.pointFeature(GeoPoint.of(gcj[0], gcj[1]), props));
    }
    return geoJsonBuilder.collection(features);
}
```

```typescript
function addHeatmapLayer(data: FeatureCollection) {
  const map = mapInstance.value; if (!map) return
  // 清理旧图层
  try { if (map.getLayer('heatmap-layer')) map.removeLayer('heatmap-layer') } catch {/* */}
  try { if (map.getSource('heatmap-signups')) map.removeSource('heatmap-signups') } catch {/* */}
  if (!data?.features?.length) return
  map.addSource('heatmap-signups', { type: 'geojson', data: data as any })
  map.addLayer({
    id: 'heatmap-layer', type: 'heatmap', source: 'heatmap-signups',
    paint: {
      'heatmap-weight': ['get', 'weight'],
      'heatmap-intensity': 1.5,
```

**校区范围边界校验（P2-AM-15, P2-AM-12/13）**

SpatialCalculator中定义矿大南湖校区GCJ-02与WGS-84两套坐标系边界常量。前端map.ts导出`CAMPUS_BOUNDS_GCJ02`给useMap.ts设置MapLibre maxBounds限制拖拽范围；MIN_ZOOM=14/MAX_ZOOM=17限制缩放级别。天地图底图使用GCJ-02坐标，数据库存储WGS-84坐标，CoordConvertService实现双向转换。

```java
// SpatialCalculator.java - 双坐标系校区边界常量
// GCJ-02 边界（天地图底图使用，以[117.140,34.215]为中心外扩约2km）
public static final double CAMPUS_GCJ02_LNG_MIN = 117.118;
public static final double CAMPUS_GCJ02_LNG_MAX = 117.162;
public static final double CAMPUS_GCJ02_LAT_MIN = 34.197;
public static final double CAMPUS_GCJ02_LAT_MAX = 34.233;

// WGS-84 边界（GPS原始坐标）
public static final double CAMPUS_WGS84_LNG_MIN = 117.112;
public static final double CAMPUS_WGS84_LNG_MAX = 117.156;
public static final double CAMPUS_WGS84_LAT_MIN = 34.199;
public static final double CAMPUS_WGS84_LAT_MAX = 34.235;

public boolean isWithinCampusGcj02(double lng, double lat) {
    return lng >= CAMPUS_GCJ02_LNG_MIN && lng <= CAMPUS_GCJ02_LNG_MAX
        && lat >= CAMPUS_GCJ02_LAT_MIN && lat <= CAMPUS_GCJ02_LAT_MAX;
}
```

**K-Means++集合点推荐**

根据活动报名学生的签到坐标，使用K-Means++初始化算法（按距离平方概率分布选择初始中心，优于朴素K-Means随机初始化）进行聚类分析，输出最优集合点位置。聚类在WGS-84坐标系下进行，输出结果转为GCJ-02对齐天地图显示。

```java
// MapServiceImpl.java - K-Means++聚类核心
private List<double[]> kMeansPlusPlus(List<double[]> points, int k) {
    List<double[]> centroids = new ArrayList<>();
    Random rng = new Random();
    centroids.add(points.get(rng.nextInt(points.size())).clone());
    // 按"距离最近中心的平方距离"概率分布选取后续中心
    for (int c = 1; c < k; c++) {
        double[] distSq = new double[points.size()];
        double total = 0;
        for (int i = 0; i < points.size(); i++) {
            double minD = Double.MAX_VALUE;
            for (double[] cent : centroids) {
                double d = Math.pow(points.get(i)[0]-cent[0], 2)
                         + Math.pow(points.get(i)[1]-cent[1], 2);
                if (d < minD) minD = d;
            }
            distSq[i] = minD; total += minD;
        }
        double r = rng.nextDouble() * total;
        double cum = 0; int chosen = 0;
        for (int i = 0; i < points.size(); i++) {
            cum += distSq[i]; if (cum >= r) { chosen = i; break; }
        }
        centroids.add(points.get(chosen).clone());
    }
    // Lloyd迭代(30轮)优化中心位置
    for (int iter = 0; iter < 30; iter++) {
        // ... 分配点到最近中心 + 重新计算中心坐标
        // 直到收敛(all centers unchanged)或达到迭代上限
    }
    return centroids;
}
```

**3.4 志愿足迹地图**

**功能概述**：为每位学生提供个人志愿活动时空回顾功能，在地图上按时间顺序展示所有签到过的活动位置，通过彩色圆点标注并用连线按时序串联，形成"志愿足迹轨迹"。同时在下方面板中展示每次活动的详情卡片（活动名称、签到时间、志愿时长、GPS坐标）和统计面板（总参与次数、覆盖地点数、累计服务时长）。

**操作步骤（3步大功能）**：

（1）**进入足迹页面**：学生点击导航栏"志愿足迹"菜单，进入MyFootprint.vue页面。页面加载时调用`GET /api/signups/my-footprint`接口，后端查询当前用户所有签到记录（sign_in_lng和sign_in_lat非空），按sign_in_time升序排列，关联activity表获取活动标题。

（2）**地图时空轨迹展示**：前端收到足迹数据后，调用wgs84ToGcj02()将WGS-84签到坐标转为GCJ-02。使用MapLibre的line layer（蓝色半透明）将全部足迹点按时间顺序连接成轨迹线。使用circle layer渲染点位：第一次活动用蓝色大圆点（radius=10）、最后一次用绿色圆点、中间活动用灰色圆点，直观体现志愿历程的时间演化。

（3）**点击查看活动详情**：绑定MapLibre click事件到circle layer。点击任意足迹点后，从GeoJSON properties中提取activityId、signInTime、volunteerHours、WGS-84坐标等信息，在下方el-descriptions卡片中展示活动详情。同时页面上方显示统计面板：总参与次数、覆盖地点数、累计服务时长。

```java
// SignupServiceImpl.java - 足迹数据查询
public List<Map<String, Object>> getFootprintData(Long userId) {
    List<Signup> signups = list(new LambdaQueryWrapper<Signup>()
            .eq(Signup::getUserId, userId)
            .isNotNull(Signup::getSignInLng).isNotNull(Signup::getSignInLat)
            .orderByAsc(Signup::getSignInTime));
    Set<Long> activityIds = signups.stream()
            .map(Signup::getActivityId).collect(Collectors.toSet());
    Map<Long, String> titleMap = new HashMap<>();
    if (!activityIds.isEmpty())
        activityMapper.selectBatchIds(activityIds)
            .forEach(a -> titleMap.put(a.getId(), a.getTitle()));
    return signups.stream().map(s -> {
        Map<String, Object> m = new LinkedHashMap<>();
        m.put("activityId", s.getActivityId());
        m.put("activityTitle", titleMap.getOrDefault(s.getActivityId(), "未知活动"));
        m.put("lng", s.getSignInLng()); m.put("lat", s.getSignInLat());
        m.put("signInTime", s.getSignInTime() != null ? s.getSignInTime().toString() : null);
        m.put("volunteerHours", s.getVolunteerHours());
        return m;
    }).collect(Collectors.toList());
}
```
