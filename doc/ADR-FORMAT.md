# ADR 格式

ADRs 存放在 `docs/adr/` 目录中，使用顺序编号：`0001-slug.md`、`0002-slug.md` 等。
按需创建 `docs/adr/` 目录——只在需要第一个 ADR 时才创建。

## 模板

```markdown
# {序号}. {标题}

## 状态

{提议中 | 已接受 | 已弃用 | 已替代}

## 上下文

{描述问题以及促成该决策的技术背景。}

## 决策

{我们选择了什么？}

## 理由

{为什么选择了这个方案？包含权衡分析。}
```

## 编号规则

- 从 `0001` 开始递增
- slug 使用小写 kebab-case（如 `use-postgis-for-spatial-queries`）
- 编号固定为 4 位数字，前面补零

## 更新已有 ADR

- 如果 ADR 被弃用或替代，更新其状态为"已弃用"或"已替代（由 000x 替代）"
- 不要删除已弃用的 ADR，保留历史记录
