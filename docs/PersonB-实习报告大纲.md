# Person B 实习报告大纲 — 活动与地图子系统

> 基于GIS的校园志愿活动服务系统 | 中国矿业大学 | 2026年7月

---

## 一、项目总览

### 1.1 项目定位
面向矿大南湖校区的志愿活动 Web 平台，学生可在地图上浏览、报名志愿活动并 GPS 签到签退。

### 1.2 技术架构

```
前端: Vue 3 + TypeScript + Element Plus + MapLibre GL JS + 天地图 WMTS
后端: Spring Boot 3.2 + MyBatis-Plus + Spring Security 6 + JWT
数据库: MySQL 8.0
GIS服务: GeoServer WFS + PostGIS + QGIS
```

### 1.3 子系统划分（6模块单体架构）

| 子系统 | 缩写 | 职责 | 负责人 |
|--------|------|------|:--:|
| 用户与权限 | UPM | 注册/登录/JWT/角色管理/组织者申请审批 | A |
| **活动创建与审核** | **ACA** | **活动CRUD/状态流转/模板/组织者仪表盘** | **B** |
| 活动地图 | AM | GeoJSON/GeoServer图层/边界限制/热力图 | C |
| 报名管理 | ABM | 报名/取消/名额/审核 | B |
| 签到签退 | APM | GPS签到/二维码/围栏/时长计算 | C |
| GIS公共模块 | GEO | Haversine/坐标转换/空间校验 | C |

### 2.4 技术亮点

1. **MapPicker 地图选点**：自建 MapLibre 实例，独立于 BaseMap，彻底解决跨组件 ref 解包 + GCJ-02/WGS-84 双向坐标转换
2. **AI 功能接入**：Pollinations.ai 免费 API，零配置零费用，三级降级（API→Ollama→Mock），同学拉取即用
3. **活动编辑分级权限**：草稿全字段可编 / 已发布仅描述封面 / 已结束禁止编辑
4. **封面图全链路**：前端上传→后端 UUID 存储→Mock 内联 SVG 降级→AI 生成
