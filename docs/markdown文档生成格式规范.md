# 项目文档 Markdown 生成格式规范

> **用途**：统一本项目中由 LLM 生成的所有 Markdown 文档的结构与风格，确保多人协作与多轮生成的一致性。
> **适用范围**：需求分析报告、系统设计文档、技术方案、开发计划等所有项目文档。

---

## 1. 标题层级规范

文档采用四级标题体系，较少使用五级：

| 层级 | 语法 | 用途 | 示例 |
|------|------|------|------|
| 一级 `#` | `# 标题` | 文档主标题、章节编号（一、二…） | `# 一、引言` |
| 二级 `##` | `## 标题` | 子章节编号（1.1、1.2…） | `## 1.1 编写目的` |
| 三级 `###` | `### 标题` | 细分子章节（2.2.1…） | `### 2.2.1 硬件环境` |
| 四级 `####` | `#### 标题` | 用例/功能点名称 | `#### 用例 AM-01：活动地图浏览` |

**规则：**
- 一级标题为文档模块级分割，章节序号使用中文数字（一、二、三…）。
- 二级标题使用阿拉伯数字两级编号（1.1、2.3）。
- 三级标题使用阿拉伯数字三级编号（2.2.1、5.3.2）。
- 四级标题一般不再编号，用途例名直接作为标题。

## 2. 表格规范

### 2.1 标准表格

```markdown
| 列1 | 列2 | 列3 |
|------|------|------|
| 内容 | 内容 | 内容 |
```

### 2.2 用例表格（核心格式）

用例是本项目最重要的文档元素，采用固定列结构：

```markdown
#### 用例 XX-01：用例名称

| 项目 | 说明 |
|------|------|
| **用例名称** | xxx |
| **用例编号** | XX-01 |
| **参与者** | 角色名称 |
| **前置条件** | 条件描述 |
| **后置条件** | 结果描述 |

**主事件流**：
1. 步骤1描述。
2. 步骤2描述。
3. …

**异常事件流**：
- **E1 异常名称**：异常描述与处理方式。
- **E2 异常名称**：异常描述与处理方式。

**替代事件流**：（可选）
- **A1 替代场景**：替代流程描述。
```

### 2.3 对比/说明表格

强调列别名的使用：

```markdown
| 术语 | 定义与解释 |
|------|-----------|
| 术语A | 定义内容 |
```

## 3. 列表规范

- **无序列表**使用 `- `（短横+空格），用于罗列性质内容。
- **有序列表**使用 `1. ` 格式，用于步骤性内容（如主事件流）。
- 列表嵌套缩进使用 2 个空格。
- 列表项内如需多行，续行对齐第一行文本。

```markdown
- 一级条目
  - 二级条目
    - 三级条目
```

## 4. 引用与标注规范

### 4.1 图注

图表统一使用引用块标注：

```markdown
> **图N 图表名称**
```

图表本身不嵌入 Markdown 中时，加说明文字：

```markdown
> **图1 顶层数据流图**（图表详见完整版报告）
```

### 4.2 内联代码与代码块

- 文件名、路径、API端点、配置键名 使用内联代码 `` ` `` 包裹：`` `activity` 表 ``、`` `ST_Distance_Sphere` ``
- 多行代码使用围栏代码块并标注语言：

```sql
CREATE SPATIAL INDEX idx_activity_location ON activity(location_point);
```

## 5. 分隔与空行规范

- 一级标题（章节）之间使用 `---` 分隔线。
- 每个标题上下各保留一个空行。
- 表格上下各保留一个空行。
- 列表上下各保留一个空行。

## 6. 内容风格规范

### 6.1 术语一致性

全文使用统一术语，不在不同章节混用同义词：

| 统一用词 | 避免使用 |
|---------|---------|
| 活动组织者 | 组织方、发布者、发起人 |
| 学生用户 | 普通用户、学生端、参与者 |
| 学院审核员 | 院级管理员、初审员 |
| 学校管理员 | 校级管理员、系统管理员 |

### 6.2 编号体系

- 用例编号格式：`子系统缩写-序号`，如 `AM-01`、`ACA-02`。
- 子系统缩写对照：

| 缩写 | 全称 |
|------|------|
| AM | 活动地图子系统 |
| ACA | 活动创建审核子系统 |
| ABM | 活动报名管理子系统 |
| APM | 活动过程管理子系统 |
| UPM | 用户与权限管理子系统 |

### 6.3 技术名词格式

- 首次出现的核心技术名词可加粗：**天地图API**、**Spring Boot**
- 技术栈统一用完整名称：`Spring Boot 3.2` 而非 `SB 3.2`
- 坐标系与标准号直接写：`WGS-84（EPSG:4326）`

## 7. 文档头部元信息模板

需求分析/设计类文档统一使用以下头部：

```markdown
# 文档标题

**文档类型**（如：需求分析报告）

单位/组织名称

日期（格式：YYYY年MM月）
```

## 8. 附录格式

附录使用固定结构：

```markdown
# 附录X：附录标题

> 图表

## X.1 子附录标题

| 列1 | 列2 | 列3 |
|-----|-----|-----|
```

## 9. 分工表格

文档末尾使用固定列结构：

```markdown
| 姓名 | 学号 | 负责内容 |
|------|------|---------|
| 姓名 | 学号 | 章节名称（以、分隔） |
```
